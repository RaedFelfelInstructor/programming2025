## 📋 Overview

So far, all your code has lived inside `Main()` — one long sequence of instructions. This week, you'll learn how to **break your programs into smaller, reusable pieces** called methods. Methods are the foundation of clean, organized code. They let you name a block of logic, call it whenever you need it, and avoid writing the same code twice.

Think of methods like recipes in a cookbook: each recipe (method) has a name, a list of ingredients (parameters), and produces a result (return value). You can follow any recipe whenever you need it, without rewriting the instructions each time.

---

## 🎯 Learning Objectives

By the end of this week, you will be able to:

1. **Explain** why methods are essential for writing maintainable code
2. **Define** methods with appropriate return types, names, and parameters
3. **Call** methods and use their return values in expressions
4. **Distinguish** between `void` methods and value-returning methods
5. **Create** overloaded methods with different parameter lists
6. **Describe** how the call stack works at a basic level
7. **Refactor** a long program into well-organized methods
